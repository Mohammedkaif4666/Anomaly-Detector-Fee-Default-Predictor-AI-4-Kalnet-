Platform Features

Everything a School Needs
to Act Before It's Too Late

Six modules working together: from raw data generation to real-time dashboard alerts — fully automated, no external APIs needed.

🔍

Attendance Anomaly Detection

IsolationForest identifies students whose attendance patterns are statistically abnormal — catching sudden drops before they become crises. Achieves 63% recall on real anomalous students.

💰

Fee Default Prediction

GradientBoosting trained on historical term payment data predicts which students will default next term — with ≥70% recall on the rare default class.

📊

Real-Time Dashboard

Live student table with search, filter, sort, and pagination. Click any student to see their full AI-generated risk analysis and 30-day attendance chart.

🗄️

500 Students · 72K Records

Realistic synthetic dataset: 500 students across 10 classes, 72,000 daily attendance records, 1,500 fee payment records with income brackets and family data.

⚡

FastAPI Backend

All ML predictions pre-computed at startup and cached in memory — instant dashboard load with zero inference latency per request. Full REST API with Swagger docs.

🛡️

No Data Leakage

Feature engineering strictly uses Terms 1 & 2 historical data to predict Term 3 — preventing the trivial pattern that would make probabilities always 0% or 100%.

Pipeline

From Raw Data to
Actionable Alerts

Four automated stages take you from data generation to a live dashboard — run once, then just start the server.

1

Generate Data

Mohammed Kaif's generator creates 72K attendance records and 1.5K fee records with realistic anomalous patterns injected for 14% of students.

2

Engineer Features

Rohith Koppu transforms raw logs into 10 ML features: absence streaks, payment history averages, income brackets, and more — with quality validation.

3

Train ML Models

Om trains IsolationForest for attendance. Are Samhith trains GradientBoosting for fees — with threshold tuning to guarantee ≥70% recall on defaults.

4

Serve Dashboard

Jyothsna's FastAPI loads both models at startup, pre-computes all 500 students, and serves the live dashboard + REST API on a single server command.
